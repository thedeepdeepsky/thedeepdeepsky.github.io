二钥 efile 浏览器加密器

本页面离线运行；用于对标 02.py。它会读取 Root CA / signer / recipient，自动为本次 efile 生成一套 authority/auth key + cert，然后输出 .efile 和三个 decrypt_mobile.html 兼容文本包。

浏览器版只生成完整 .efile，不生成 Python 版 sidecar .efile.fec 纠错文件。

1. 证书文字 bundle，可替代下面的文件选择

支持 [root_ca.key]、[root_ca.cert]、[signer.key]、[signer.cert]、[recipient.cert]。[recipient.key] 可选：提供后会被写入解密主 bundle；不提供也可以加密，解密时再单独提供 recipient 私钥。

2. 或选择 PEM 文件
3. 私钥口令和 auth 参数
4. 待加密内容
5. 单独 efile 文本包

只包含 [efile]，方便单独发送或粘贴到解密页的 .efile 文本包输入框。

6. authfile + efile bundle

包含 [efile]、[authority.key]、[authority.cert]。适合把本次 efile 和对应 auth files 作为一组单独保存。

7. 解密全套 bundle

给 decrypt_mobile.html 一次性粘贴使用：[efile]、[root_ca.cert]、[recipient.cert]、[authority.key]、[authority.cert]。如果输入了 [recipient.key]，这里也会一并输出；否则解密方需要再单独提供 recipient 私钥。
