二钥 efile 证书生成器

本页面离线运行；用于替代 01.py 的证书生成部分。生成 Root CA、recipient、signer 三组 key/cert。authority/auth 仍由加密页按每个 efile 自动生成。

请妥善保存 Root CA 私钥和 signer 私钥。泄露 Root CA 私钥会导致别人可签发新的 auth 证书。

1. 密码设置（对标 01.py，三个密码分开）
2. 证书参数
3. 全量证书文字 bundle

管理员/总备份使用。包含 [root_ca.key]、[root_ca.cert]、[signer.key]、[signer.cert]、[recipient.key]、[recipient.cert]。

4. 加密方需要提前留存的文字 bundle

给加密页 enc.html 使用。默认不包含 [recipient.key]，只包含加密和签发本次 auth 证书所需内容：[root_ca.key]、[root_ca.cert]、[signer.key]、[signer.cert]、[recipient.cert]。

5. 解密方文本包

给解密方提前保存。包含解密 K1 所需的 [recipient.key]、[recipient.cert]，以及验签/校验证书链所需的 [root_ca.cert]。不包含 Root CA 私钥和 signer 私钥。
