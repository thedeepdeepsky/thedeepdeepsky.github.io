二钥 .efile 浏览器解密器

本页面离线运行；文件和私钥不会上传。支持 Python 版 01.py 生成的 ENCRYPTED PRIVATE KEY：PBES2 + PBKDF2-HMAC-SHA256 + AES-256-CBC。

HTML 版当前只处理完整 .efile。如果 .efile 已损坏且需要 sidecar .efile.fec 纠错，请先用 Python 版 03.py。

1. 加密文件

支持 [efile]:、[locked.efile]: 等分段；如果这里填写，会优先于文件选择框。

2. 信任根和两把解密证书
3. 快捷 PEM 文本包，可替代上面的 PEM 文件上传

把 Root CA、recipient、authority 的证书和私钥一次性粘贴到这里。名称不要求顺序；已解析的文本会优先于文件选择框。也可以把 [efile]: 混在这里，页面会自动识别。

4. 可选：指定 signer 指纹 pinning
